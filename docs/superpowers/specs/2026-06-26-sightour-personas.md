# Sightour 用户画像文档

> **文档类型**：产品 / UX 用户画像规范
> **阶段**：MVP（Phase 1）范围
> **范围**：3 类核心用户画像，覆盖短期商务、自由行旅游、长期留学三类来华外籍人士。
> **版本**：v1.0 — 2026-06-26

---

## 0. 文档元信息

### 用途
本文档定义 Sightour MVP 的三类目标用户。Phase 1 内每个产品、设计、内容决策都应能对照这些画像自检：「这个功能服务于哪个画像？是否按画像描述的方式解决了他的问题？」

### 读者
- 产品经理（优先级仲裁）
- UX / 交互设计师（屏级决策）
- 内容编辑（哪些 POI 与提示值得关注）
- 工程师（哪些流程必须快、哪些可以慢）

### 信息源声明
**这是设计假设，不是经用户研究验证的结论。** 每类画像基于以下来源构建：

1. PRD §1.4（目标受众描述）
2. 中国入境政策与上海旅行实况的公开知识
3. 邻近产品（TripAdvisor、Google Maps、Wise、Airbnb 房东端）的用户模式
4. 2024–2025 期间英语旅行社区（Reddit r/China、r/shanghai、YouTube 旅行博主）的高频吐槽点

**MVP 上线前**，需通过以下方式验证：

- 每类画像 5–8 次用户访谈（在目标来源地：法兰克福、旧金山、雅加达）
- 前 1,000 次下载的埋点数据，确认主导流程路径
- 应用内纠错提交按使用模式聚类分析

### 如何使用本文档
- 提新功能时：识别它服务于哪些画像、对应痛点的严重度、命中情绪曲线的哪个阶段。
- 提删除功能时：找到它落入哪个画像的「永远不会用」场景。
- 排路线图时：按（服务画像数 × 痛点严重度 × 频次）加权排序。

### 不在本文档范围内
- 详细量化规模（TAM、MAU 目标）— 由商业策略文档负责。
- 行为数据（热力图、漏斗转化）— 上线后才有。
- A/B 测试结果 — 由实验框架负责，框架尚未建立。

## 1. 画像 · Emma Becker

**商务访客 · 德国 · 法兰克福 · 28 岁**

```
┌──────────────────────────────────────────────┐
│  [photo]                                       │
│                                               │
│  "Drei Tage Messe, ein Tag Kunde. Ich will    │
│   nur wissen, wie ich heute vom Flughafen     │
│   zum Hotel komme."                           │
│                                               │
│  — 三天展会加一天客户拜访。我只想知道今天    │
│    怎么从机场到酒店。                         │
└──────────────────────────────────────────────┘
```

### 背景
Emma 是一家德国中型工业自动化公司的中级产品经理。她所在的公司已连续五年参加中国国际工业博览会（上海），她个人出席过两次。这是她第三次来中国。她能讲够用的英语（与国际客户的工作语言），但完全不会中文。她对手机应用很熟练，在欧洲使用 Apple Pay 和 Google Pay，日程管理用 Outlook。

她持德国护照（申根 → 自 2024 年中国单方面免签政策起可免签停留 30 天）。本次在上海停留四天：周一傍晚抵达，周二至周四参展，周五上午拜访客户，周五下午飞回。

机票和酒店由公司差旅系统（汉莎 / HRS）统一预订，因此 Sightour 不需要帮她订机票酒店。公司报销上限为 €120/晚对应 4 星 — 她最终住在浦东万豪或类似酒店。

### 典型行程
- **第 0 天（周一傍晚）** ✈ LH 720 法兰克福 → 浦东，17:50 落地
  - 公司已确认机场接送，但她收到的确认单是中文
  - *★ 痛点：看不懂机场接送说明*
- **第 1 天（周二上午）** 08:00 国家会展中心开馆
  - 乘酒店班车去展馆
  - 午餐：展馆食堂（外卡受理时好时坏）
  - 晚间：本地经销商在浦西组织的私房菜晚宴
- **第 2 天（周三）** 全天参展；下班后与德国同事喝一杯
- **第 3 天（周四）** 上午参展 + 下午飞回法兰克福
- **第 3 天傍晚取消** — 一位德国大客户要求周四上午简短会面 → 公司为她延期 2 晚
  - **★ 现在她需要：紧急改订酒店、浦东附近的双语会面场所、签证延期确认（她的 30 天没问题，但她想确认）**
- **第 4 天（周五）** 陆家嘴客户会面（不同区域）→ 需在不同会面点之间穿梭
  - **★ 痛点：如何叫到正规出租车（不打表骗局）、如何用外卡支付午餐**
- **第 5 天（周六）** 飞回家

### 三大目标
1. **在最初 90 分钟内** 从机场抵达酒店，可预期、不需要打德国客服电话。
2. **按时参加所有预定会面**，不出物流事故损害她在客户前的可信度。
3. **全程与德国团队和家里保持联系** — WhatsApp、邮件、手机数据。

### 三大痛点

| # | 痛点 | 严重度 |
| - | --- | --- |
| 1 | **在中文交通标识 / 酒店预订确认前一脸懵**——偏偏在最需要它的时刻（长途飞行后很累）。 | **5 / 5** |
| 2 | **外卡受理不稳定**——出租车、小餐厅拒卡。被迫携带大额人民币现金，而她是在机场以很差汇率换的。 | **4 / 5** |
| 3 | **行程变更**（延期、客户改时间）触发一连串改订，需要她无法拨通的双语电话。 | **3 / 5** |

### 技能 / 设备画像
- **手机熟练度**：高 — 在德国一切应用搞定（日历、银行、地图）。
- **中文**：零基础功能性中文；只能读拉丁字符。
- **数据**：开通了德国运营商（德国电信）的中国-欧洲漫游套餐 €30 / 200 MB / 天。用一天后断网。
- **她在德国常用的应用**：Google Maps、Apple Maps、Booking.com、HRS、汉莎、WhatsApp、Gmail、Outlook、DKB 银行、N26 银行、Tripadvisor（仅用于行前研究）。
- **在中国需要安装后才能用的应用**：Google Maps（偶尔能用）、WhatsApp（2023 年后免 VPN 可用但语音视频不稳）、Gmail（IMAP 可用）、Instagram / Facebook / X（被封）。
- **她落地后装的**：支付宝国际版（同事推荐）。

关键时刻：
- **压力峰值**：落地后第 5 分钟 — 一直持续到坐进酒店的车。
- **压力谷底**：第 2–4 天在展馆（一路流程化，展位说英语）。
- **隐性压力**：整段旅程围绕支付摩擦。

### 「她会爱上 APP 的瞬间」
1. 在登机口，她打开 Sightour，点选国籍，看到：「德国 → 30 天免签，无需提前申请。✓」——瞬间安心，无需发邮件问公司要政策 PDF。
2. 应用在机场给她一张「一键叫车」卡片，附带三条提前准备（「给司机看这个酒店的中文名：上海浦东万豪酒店」）——她不再手忙脚乱。
3. 客户会面改期后，她打开 APP 的「行程变更」清单（签证、住宿、现金充值、交通更新），5 分钟搞定，不用打德国客服。

### 「她永远不会用 APP 做」
- 行前订机票 / 酒店（公司差旅系统处理）。
- 一般观光推荐（公司晚宴占满所有晚上；她本次没时间玩）。
- 社交 / 社区功能（她不想和其他用户互动，要快速私人答案）。
- 关于签证政策历史的长篇内容（她要一行答案，不要 2,000 字科普）。

## 2. 画像 · James & Maria Reyes

**自由行游客夫妇 · 美国 · 旧金山 · 32（James）+ 30（Maria）岁**

```
┌──────────────────────────────────────────────┐
│  [photo: James]  [photo: Maria]                │
│                                               │
│  "Our first time in China — we want to do      │
│   the bucket-list stuff, eat real food, but   │
│   we don't want to get ripped off."           │
│                                               │
│  — 我们想看外滩、吃小笼包、坐磁悬浮，但     │
│    最大恐惧是踩进一个 ¥1,200 的烂饭馆。      │
└──────────────────────────────────────────────┘
```

### 背景
James 是 SoMa 一家 B 轮创业公司的软件工程师；Maria 是 UX 设计师。结婚两年，尚未考虑孩子，正在「亚洲大旅行」。他们曾多次去欧洲、墨西哥、日本；这是第一次来中国大陆。

James 因公去过一次香港（没进大陆）。他们看 YouTube 上海旅行视频两个月了。两人都能讲西班牙语日常对话，凑合日语；都不会中文。他们自称「吃货」—— 一半旅行预算花在餐厅。

持美国护照。30 天免签 2025 年曾短暂暂停后恢复；为稳妥起见，他们申请了标准的 10 年旅游签证（L 签）—— 计划停留 12 天，希望有弹性。

预算：约 $8,000 含机票。酒店：静安区中端精品（¥800–1,200 / 晚），通过 Trip.com 和 Booking.com 混合预订。

### 典型行程
- **第 1 天（抵达）** ✈ 旧金山 → 浦东（经台北），14:20 落地
  - 通过 Booking.com 预定了酒店接送 — 司机没来
  - *★ 他们必须在零中文、VPN 不稳的情况下，搞定机场到市区*
  - 16:00 入住静安酒店，倒头就睡
- **第 2 天** 外滩 + 豫园 + 南京路步行；晚餐在他们 YouTube 上看到的著名小笼包店
  - **★ 痛点：名店排队 90 分钟，且有独立的「游客菜单」价格贵 3 倍**
- **第 3 天** 法租界漫步 + 田子坊 + 黄浦江夜游
- **第 4 天** 朱家角水乡一日游（需自行去长途汽车站购票）
- **第 5 天** 上海博物馆 + 上海中心观景 + ¥600 的私房菜晚宴
  - **★ 痛点：通过礼宾 APP 订的餐厅是「fusion」，他们想要真正的本地馆子**
- **第 6 天** 长时间 — 醒来胃不舒服，需要药店，不确定是否要处方
  - **★ 痛点：看不懂标签的药店、不知道哪个品牌等同美国 Tylenol、不知道要不要处方**
- **第 7 天** 恢复日 + 上海迪士尼
- **第 8 天** 高铁去杭州 2 晚
- **第 9 天** 西湖，当晚返沪
- **第 10 天** 临时购物（给 James 妈妈的礼物），再游黄浦江
- **第 11 天** 武康路区域，咖啡馆文化 morning
- **第 12 天** ✈ 旧金山

### 三大目标
1. **吃到真正的本地菜**而不被宰——理想情况下是游客常规路径之外但又安全的本地馆子。
2. **理解上海的整体地理**——感到自己懂这座城是怎么拼起来的，而不是只去 N 个地标。
3. **保持健康与安全**——既不要医院体验，也不要被定义整段旅行的骗局事件。

### 三大痛点

| # | 痛点 | 严重度 |
| - | --- | --- |
| 1 | **「真本地 vs 游客陷阱」辨别**——他们分不清哪家餐厅是真的本地人去的，哪家是给游客包装的。 | **5 / 5** |
| 2 | **非连锁店的沟通障碍**——无英文菜单、无中文、肢体语言只能帮到浅层，深层菜单项目无法确认。 | **5 / 5** |
| 3 | **支付摩擦**——他们的 Chase Sapphire 在酒店和大餐厅能用，但在半数咖啡馆和小店被拒；最后手上留 ¥3,000 现金不知怎么花。 | **4 / 5** |

### 技能 / 设备画像
- **手机熟练度**：极高 — 地图、餐饮、银行应用重度用户。
- **中文**：零，能读拼音但不能读汉字。
- **数据**：买了 T-Mobile 15 天高速套餐（$50）。地图 / 消息能用，但 Google Maps 即便漫游也不稳。
- **手机上已有的应用**：Google Maps、Apple Wallet、Tripadvisor、Yelp（无上海覆盖）、Uber（无服务）、Booking.com、Agoda、WhatsApp（可用）、Instagram、TikTok 国际版（非抖音）。
- **抵达后安装**：Trip.com、支付宝（被机场二维码强迫）、DiDi 国际版、Pleco（带 OCR 的中文字典）、VPN。
- **货币舒适度**：信用卡舒服，现金不舒服，不信任机场换汇柜台（汇率差确实坑）。

      │   ╱  ╲          ╱╲       ╱─╲
      │  ╱    ╲       ╱   ╲    ╱    ╲

      │       ╲╱╱       ╲╱╲╱        ╲
      └────────────────────────────────

关键时刻：
- **第 2 天晚饭排队** 是决定 Sightour 是否成为他们后续每天随行 APP 的关键时刻。
- **第 6 天药店** 是整段旅行压力最高的时刻。
- **第 7 天起** 他们要么已形成使用习惯（高 APP 使用），要么已放弃（不再需要）。

### 「他们会爱上 APP 的瞬间」
1. 第 2 天，著名小笼包店排队太长，他们点「餐饮」筛选，10 分钟步行内找到三家口碑相当的店，每家标着「✓ 已核验 · 有英文菜单 · 支持外卡」。
2. 第 6 天，其中一位胃不舒服，他们打开 Sightour → 工具 → 常用语 → 医疗，找到「我对 X 过敏 / 我要看医生 / 最近的药店在哪？」的准确中文句，10 分钟解决。
3. 第 7 天，整理好的「杭州高铁一日游」攻略告诉他们虹桥站哪个站台、车座选哪侧风景更好看、什么值得跳过。

### 「他们永远不会用 APP 做」
- 订机票或酒店（已在 Booking.com、Trip.com 完成）。
- 西餐或连锁餐厅（自己能找）。
- 跟团游（他们想独立）。
- 夜生活 / 酒吧评价（不太喝，也不信网上评价）。
- 个人照片日志（Instagram 已经做了）。

## 3. 画像 · Lin Wei

**长期留学生 · 印度尼西亚 · 雅加达 → 上海 · 19 岁**

```
┌──────────────────────────────────────────────┐
│  [photo]                                       │
│                                               │
│  "Saya harus urus SIM card, rekening bank,     │
│   dan tempat tinggal dalam dua minggu.        │
│   Mana yang bisa diurus sendiri?"            │
│                                               │
│  — 我要在两周内搞定理 SIM 卡、银行账户、住   │
│    房。哪些能自己搞定，哪些必须去政府窗口    │
│    排队？                                      │
└──────────────────────────────────────────────┘
```

### 背景
Lin Wei 即将在上海某大学开始 4 年本科国际商务专业（学校类似：上海对外经贸大学、上海理工大学、上海外国语大学这类中等层次）。她 2026 年 5 月在雅加达高中毕业；这是她第一次独立海外生活。学费由父母支付；每月零花钱 ¥3,000，日常开支自理。

她英语流利（高中双语学校，读英文小说，IELTS 7.5）。完全不会中文 — 但计划到中国后学。她是数字原住民（在雅加达用 Grab / Gojek / Shopee 一站式），用 TikTok、Instagram、LINE、WhatsApp、Notion。她习惯了在印尼政府窗口排队办 KTP、SIM、NPWP，预期在中国也是这样，焦虑哪些能走 APP。

持学生签证（X1 签）入境，提前 2 个月在雅加达中国领事馆申请。单次入境，有效期 1 年，要求入境后 30 天内申请居留许可（居留许可）— 硬性法律死线。

### 典型行程

**行前（雅加达，提前 2 周）**
- 护照贴上 X1 签证 + JW202 表（黄色长表，关键文件）
- 大学发 WeChat 群邀请 + 50 页英文版 PDF 指南
- *★ 痛点：PDF 里哪些章节对她这种情况真正有用？*

**第 1 周（上海）**
- 第 1 天：大学联络员机场接机（X1 签持有人强制）
- 第 2 天：指定医院体检（办居留许可必需）
- 第 3-4 天：搬进已由学校预订的宿舍
- 第 5 天：派出所登记入住（24 小时死线 — 入住后必须 24h 内登记）
- 第 6 天：在 ICBC 开银行账户（需要学校介绍信 + 护照 + 居留许可申请回执）
- 第 7 天：开学第一课
- *★ 痛点：每个政府窗口都只说中文，遇到会英文的工作人员算运气*

**第 2-3 周**
- 在上海公安局出入境管理局（PSB / 出入境管理局）申请居留许可
- 办理本地手机套餐（中国移动，学生套餐约 ¥50/月）
- 找下学期校外长期住房
- *★ 痛点：用中文签租房合同，分不清什么是正常的、什么是坑*

**第 1-2 月**
- 安顿下来，找常用超市、清真食物（她是穆斯林）
- 开始找兼职实习
- *★ 痛点：清真食物 — 大多数地图应用默认显示猪肉为主的菜单，没有饮食过滤*

**第 3-12 月**
- 偶尔的续签（居留许可第 11 个月续期）
- 签证相关紧急情况（丢护照、家属来访、学校假期出行）
- *★ 痛点：每次续签都要重新搞清楚「这次要带什么」*

### 三大目标
1. **30 天内办下居留许可（居留许可）** — 错过死线会导致罚款、遣返、甚至多年禁止入境。
2. **搞定无聊的基础设施**（银行、SIM、手机、交通、住房），不被骗或多花冤枉钱。
3. **找到一个社群** — 至少一个朋友、一家她信得过的清真餐厅、一个知道她点单的咖啡馆。

### 三大痛点

| # | 痛点 | 严重度 |
| - | --- | --- |
| 1 | **有硬性死线的官僚流程**（24h 派出所登记、30 天居留许可、年度续签）— 任何一环错过都连锁罚款或更糟。信息存在但散落在 PDF、微信群、口耳相传里。 | **5 / 5** |
| 2 | **清真食物 + 饮食过滤** — 大多数地图应用不能让她过滤掉猪肉 / 非清真选项。看菜单对她当前而言不可能。 | **4 / 5** |
| 3 | **租房与金融服务的骗局风险** — 印尼有类似骗局文化，所以她很警觉，但她不知道中国平台上哪些是 Tokopedia / Gojek 的合法对标。 | **4 / 5** |

### 技能 / 设备画像
- **手机熟练度**：极高（数字原住民）。
- **中文**：落地时零，计划大一开始正式学。
- **数据**：行前在网上买了中国联通 eSIM（¥200 / 30 天 / 30 GB）。全程稳定。
- **手机上已有的应用**：Instagram、TikTok、WhatsApp（可用）、LINE、Notion、Grab、Shopee、Gojek、印尼移动银行（BCA / Jenius）、Google Maps（中国不好用）、Chrome。
- **她落地后装的**：微信（强制 — 所有人都用）、支付宝、DiDi、小红书（学生优惠）、Pleco（OCR 字典）、Spotify。
- **她必须亲自跑的点**：PSB（出入境）、派出所、银行（ICBC）、指定医院（体检）、大学行政办公室、移动运营商门店。

关键时刻：
- **第 1-5 天（行政周）**：最焦虑，每次跑腿都有死线，每个窗口都说中文。
- **第 2-3 周（中适应）**：开始找到节奏，但新问题浮现（住房、食物）。
- **第 3 月+**：已安顿，但总有一个新官僚任务在不远处。

### 「她会爱上 APP 的瞬间」
1. 第 1 天落地，应用「我刚到」的清单按顺序告诉她每个政府触点、每处需要的文件、是否需预约还是直接去。
2. 第 5 天，她要去派出所登记，找到「我要登记我的住址」准确中文句 + 文件清单 + 24 小时死线提醒。
3. 第 2 个月，她需要清真食物，POI 列表有饮食过滤（清真 / 素食 / 无猪肉）——这功能在 Google Maps 没有。

### 「她永远不会用 APP 做」
- 课表 / 学术内容（大学自己的应用处理）。
- 即时通讯（只用微信）。
- 长期储蓄 / 投资产品（印尼银行处理她的家庭转账）。
- 在亚洲的旅游签证（她的中国 X1 签不覆盖，得用别的 APP）。

## 附录 C · 待后续产品讨论的开放问题

本文档故意保留以下未决问题，需独立会话处理：

1. **Emma 的「延期」场景** — 这种情况常见吗（比如 30% 的商务访客会延期？），需要专门 UI，还是可以放在通用清单里？
2. **James & Maria「非主流路径」意图** — 他们想要独立小众推荐，还是只想要可信的主流推荐？这影响 200 个 POI 的策展策略。
3. **Lin Wei 的清真过滤颗粒度** — 「仅清真」够用吗，还是需要子类（清真认证 / 清真友好 / 素食 / 纯素 / 仅无猪肉）？
4. **跨画像机会：「东道主」用法** — 如果 Lin Wei 与 James & Maria 成了朋友，APP 是否支持邀请 / 分享流？（超出 MVP 范围但值得标记）
5. **国内二线城市出行** — 当前原型仅覆盖上海。第二个城市何时添加？这影响内容团队规模化。
